# Customer Churn Prediction

A complete, end-to-end data analytics project that predicts which telecom
customers are likely to churn (cancel their subscription), using Python,
pandas, and scikit-learn.

This project was built as a data-analyst portfolio piece and follows a
realistic workflow: data generation/loading → exploratory data analysis →
feature engineering → model training and evaluation → prediction on new data.

## Why this project?

Customer churn is one of the most common real-world problems data analysts
and data scientists are asked to solve — it directly connects to revenue,
and it's a great showcase of the full analytics skill set: SQL/pandas-style
data wrangling, visualization, statistical reasoning, and basic ML.

## Dataset

The dataset mirrors the structure of the well-known **IBM Telco Customer
Churn** dataset (customer demographics, account/contract info, and monthly
charges). Since this environment doesn't have direct access to download it,
`src/generate_data.py` generates a synthetic dataset of 3,000 customers with
the same schema and realistic, non-random relationships — for example,
month-to-month contracts, short tenure, and lack of tech support all
genuinely increase churn probability, mirroring real telecom churn drivers.

> **Using the real dataset instead:** if you have internet access, download
> the [IBM Telco Customer Churn dataset](https://www.kaggle.com/datasets/blastchar/telco-customer-churn)
> and save it as `data/customer_churn.csv` with matching column names — every
> other script will work unchanged.

**Features:**

| Column | Description |
|---|---|
| `customer_id` | Unique customer identifier |
| `gender`, `senior_citizen`, `partner`, `dependents` | Demographics |
| `tenure_months` | Months as a customer |
| `contract` | Month-to-month / One year / Two year |
| `internet_service` | DSL / Fiber optic / No |
| `online_security`, `tech_support` | Yes / No |
| `paperless_billing`, `payment_method` | Billing details |
| `monthly_charges`, `total_charges` | Billing amounts |
| `churn` | Target: Yes / No |

## Approach

1. **EDA** (`src/eda.py`) — churn rate breakdown, churn by contract type,
   tenure and monthly-charges distributions, correlation heatmap.
2. **Feature engineering** — one-hot encoding of categorical variables,
   standard scaling for the linear model.
3. **Modeling** (`src/train_model.py`) — trains and compares:
   - **Logistic Regression** (interpretable baseline)
   - **Random Forest** (captures non-linear interactions, handles class
     imbalance via `class_weight="balanced"`)
4. **Evaluation** — accuracy, precision, recall, F1, and ROC-AUC, plus
   confusion matrices and a feature-importance chart.
5. **Prediction** (`src/predict.py`) — loads the saved model and scores a
   new customer record.

## Results

On the held-out test set, the Random Forest model was selected as the best
performer by ROC-AUC. Exact numbers will vary slightly by run/seed, but on
this dataset:

| Model | Accuracy | Precision | Recall | F1 | ROC-AUC |
|---|---|---|---|---|---|
| Logistic Regression | ~0.74 | ~0.55 | ~0.33 | ~0.41 | ~0.73 |
| Random Forest | ~0.70 | ~0.46 | ~0.50 | ~0.48 | ~0.74 |

The Random Forest trades a little precision for meaningfully better recall —
in a churn use case, catching more true churners (even with some false
positives) is usually worth it, since the cost of a missed at-risk customer
is higher than the cost of an unnecessary retention offer.

The top drivers of churn (see `reports/figures/feature_importance.png`) are
**tenure, monthly charges, contract type, and internet service type** —
consistent with well-documented real-world churn behavior.

## How to run it

```bash
# 1. Clone the repo
git clone https://github.com/<your-username>/customer-churn-predictor.git
cd customer-churn-predictor

# 2. Set up a virtual environment (optional but recommended)
python3 -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Generate the dataset
python src/generate_data.py

# 5. Run exploratory data analysis (saves charts to reports/figures/)
python src/eda.py

# 6. Train and evaluate the model
python src/train_model.py

# 7. Predict on a new customer
python src/predict.py
```
